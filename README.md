# MT5 Trade Notifier

Приложение для отправки уведомлений о торговых событиях из MetaTrader 5 и TradingView в Telegram.

## Настройка перед началом работы

1. **Запустите приложение**  
   - Из папки сборки: `MT5 Trade Notifier.exe` (из папки `dist\MT5 Trade Notifier\`).  
   - Или при наличии Python: `start_notifier.bat` либо `python -m notifier.app`.

2. **Telegram (обязательно)**  
   - Создайте бота в [@BotFather](https://t.me/BotFather), получите токен.  
   - Узнайте свой Chat ID (например, через [@userinfobot](https://t.me/userinfobot) или отправьте боту сообщение и откройте `https://api.telegram.org/bot<TOKEN>/getUpdates`).  
   - В приложении введите **Токен бота** и **Chat ID**, нажмите **Сохранить**.  
   - Проверьте: кнопка **Тест** — должно прийти тестовое сообщение.

3. **MT5 и EA**  
   - Запустите нужные терминалы MetaTrader 5.  
   - Во вкладке **MT5 Терминалы** укажите **Путь к EA** (папка с репозиторием trading-bot-mt5, где лежат `Experts/` и `Indicators/`).  
   - Нажмите **Поиск терминалов** — появятся запущенные терминалы.  
   - Для каждого терминала без EA нажмите **Установить EA**.  
   - В MT5 добавьте советника **AO_Pattern_Bot** и индикатор **AO_Pattern_Watcher** на графики; после этого в приложении появятся «Активные EA» и начнут приходить уведомления по сделкам.

4. **TradingView (по желанию)**  
   - Включите **Webhook включен**, оставьте порт (например 80 или 8080).  
   - Нажмите **Открыть порт** в блоке Брандмауэр (может потребоваться запуск от администратора).  
   - Укажите **Публичный URL** — зависит от вашей ситуации:

   **Вариант A — VPS со статическим IP (без домена):**  
   - Используйте порт **80** (TradingView шлёт webhook только на 80 или 443).  
   - В **Публичный URL** введите `http://<ваш-IP>` (например `http://185.12.34.56`).  
   - Убедитесь, что порт 80 открыт в брандмауэре (кнопка «Открыть порт») и в панели VPS-хостера (Security Group / Firewall).  
   - В TradingView: Webhook URL → `http://185.12.34.56/webhook/tradingview`.

   **Вариант B — VPS со статическим IP и доменом (HTTPS):**  
   - Привяжите домен к IP, получите SSL-сертификат (Let's Encrypt).  
   - Настройте reverse-proxy (nginx/Caddy) на `443 → localhost:8080`.  
   - В **Публичный URL** введите `https://your-domain.com`.  
   - В TradingView: Webhook URL → `https://your-domain.com/webhook/tradingview`.

   **Вариант C — Локальный ПК (без статического IP):**  
   - Поднимите туннель: `ngrok http 8080` или `cloudflared tunnel --url http://localhost:8080`.  
   - Скопируйте полученный URL в **Публичный URL**.  
   - В TradingView: Webhook URL → `https://<tunnel-url>/webhook/tradingview`.

   **Защита секретом (рекомендуется):**  
   Webhook-эндпоинт открыт в интернет, поэтому без секрета любой может отправить ложный сигнал. Чтобы этого избежать:
   1. В приложении придумайте и введите произвольную строку в поле **Webhook секрет** (например `Mys3cretK3y!`).
   2. В TradingView-индикаторе **AO Cross & Color Markers** откройте настройки → группа **Webhook** → поле **Webhook Secret** — введите тот же самый секрет.
   3. Теперь каждый алерт от TradingView будет содержать поле `"secret"` в JSON-теле запроса, и приложение будет сравнивать его с настроенным. Если секреты не совпадают — запрос отклоняется с кодом 403.

   Секрет передаётся в теле POST-запроса, а не в URL, поэтому при использовании HTTPS (варианты B, C) он зашифрован. При варианте A (голый HTTP) секрет защищает от случайных запросов, но технически может быть перехвачен — для полной безопасности используйте HTTPS.

Настройки сохраняются в `config.json` рядом с исполняемым файлом (или в текущей папке при запуске через Python). При первом запуске файл создаётся из `config.example.json`, если его ещё нет.

## Возможности

- Уведомления об открытии/закрытии сделок, SL/TP из MT5
- Уведомления о сигналах индикаторов TradingView (через webhook)
- Скриншоты графиков MT5 (через EA) и TradingView (через Playwright)
- GUI с отображением активных MT5-терминалов и TradingView-индикаторов
- Сворачивание в системный трей

## Установка

### 1. Python-зависимости

```bash
pip install -r requirements.txt
```

### 2. Playwright (для скриншотов TradingView)

```bash
playwright install chromium
```

Если Playwright не установлен, TradingView-уведомления будут отправляться без скриншотов.

## Настройка

### Telegram-бот

1. Создайте бота через [@BotFather](https://t.me/BotFather)
2. Получите токен бота и chat_id
3. Введите их в приложении (поля "Токен бота" и "Chat ID") и нажмите "Сохранить"

### MT5

Установите EA `TelegramNotifier` на графики в MetaTrader 5. EA записывает события и скриншоты в общую папку `%APPDATA%\MetaQuotes\Terminal\Common\Files\tg_events\`.

### TradingView (webhook)

Для получения сигналов от TradingView нужна **платная подписка** (Pro/Pro+/Premium) для поддержки webhook-алертов.

#### Шаг 1: Публичный URL

Приложение запускает HTTP-сервер на локальном порту (по умолчанию 8080). Для открытия порта в брандмауэре Windows в интерфейсе есть поле ввода порта и кнопка **«Открыть порт»**. При нажатии приложение добавит правило в брандмауэр Windows для входящего TCP-соединения. Для этого может потребоваться запуск приложения **от имени администратора**; если прав недостаточно, рядом с кнопкой отобразится подсказка.

TradingView требует публичный URL для отправки webhook. Варианты:

**ngrok** (самый простой):
```bash
ngrok http 8080
```
Скопируйте полученный URL (например `https://abc123.ngrok-free.app`).

**Cloudflare Tunnel** (стабильнее):
```bash
cloudflared tunnel --url http://localhost:8080
```

#### Шаг 2: Настройка алерта в TradingView

1. Добавьте индикатор "AO Cross & Color Markers" на график
2. При необходимости задайте **Webhook Secret** в настройках индикатора (должен совпадать с секретом в приложении)
3. Правый клик на графике -> **Add Alert**
4. Condition: выберите индикатор "AO Cross & Color Markers"
5. Trigger: **Any alert() function call**
6. Actions -> **Webhook URL**: `https://<ваш-url>/webhook/tradingview`
7. Нажмите **Create**

Один алерт покрывает все сигналы со всех таймфреймов.

#### Шаг 3: Настройка в приложении

В приложении задайте:
- **Webhook включен**: включает/выключает HTTP webhook-сервер
- **Webhook порт**: порт HTTP-сервера (по умолчанию 8080)
- **Webhook секрет**: опциональный секрет для проверки подлинности запросов
- **Публичный URL**: базовый публичный адрес (ngrok/Cloudflare/IP), приложение само добавляет `/webhook/tradingview`
- **Скриншот, сек**: задержка загрузки графика TradingView перед снимком

Webhook-поля сохраняются автоматически при изменении. Кнопка **Сохранить** остается для ручного сохранения всех настроек.


## Конфигурация

Файл `config.json` (создается автоматически из `config.example.json`):

| Параметр | Описание | По умолчанию |
|---|---|---|
| `telegram_bot_token` | Токен Telegram-бота | `""` |
| `telegram_chat_id` | ID чата/группы | `""` |
| `common_files_path` | Путь к Common/Files MT5 | авто |
| `poll_interval_sec` | Интервал опроса папки событий | `2` |
| `heartbeat_timeout_sec` | Таймаут для статуса "warn" | `60` |
| `heartbeat_dead_sec` | Таймаут для статуса "dead" | `120` |
| `processed_retention_days` | Хранение обработанных файлов | `7` |
| `webhook_enabled` | Включить webhook-сервер | `true` |
| `webhook_port` | Порт webhook-сервера | `8080` |
| `webhook_secret` | Секрет для проверки запросов | `""` |
| `webhook_public_url` | Базовый публичный URL webhook | `""` |
| `chart_screenshot_wait_sec` | Ожидание загрузки графика | `5` |
